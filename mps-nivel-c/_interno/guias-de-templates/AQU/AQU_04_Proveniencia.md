# Tabela de Proveniência — Aquisição

> Origens: **[GSD]** / **[Jira]** / **[Manual]** / **[Derivado]**. Aquisição é processo organizacional/administrativo — quase tudo é **[Manual]** (vive em RH/gestão, não nos artefatos de engenharia). **(confirmado)** = comprovado no diagnóstico; **(hipótese)** = depende da Timeware.

| Campo | Origem | Fonte exata | Confiança |
|---|---|---|---|
| 0. Identificação | [Manual] | gatilho ao gestor | confirmado (administrativo) |
| 1. Necessidade de aquisição | [Manual] | gatilho — disparada pelo fechamento de projeto sem recurso | confirmado |
| 2. Critérios de avaliação | [Manual] | definidos pelo gestor conforme requisitos técnicos do projeto | confirmado |
| 2. Busca no mercado | [Manual] | processo de recrutamento (fora das ferramentas de engenharia) | confirmado |
| 3. Avaliação técnica e seleção | [Manual] | registro de avaliação técnica (RH/gestão) | confirmado |
| 4. Acordo (contrato PJ) | [Manual] | contrato PJ (jurídico/administrativo) | confirmado |
| 5. Monitoramento do desempenho | [Manual]+[GSD] | acompanhamento do gestor + desempenho observável nos artefatos do projeto | confirmado |
| 6. Revisão técnica das entregas | [GSD] | registros de V&V (revisão por pares no PR) + acompanhamento do projeto | confirmado |
| 6. Aceite formal de entrega | [Manual] | A DEFINIR — critério/momento ainda não estabelecido | lacuna explícita |
