# Mini-manual — Medição

**O que é.** O processo que coleta, armazena, analisa e relata dados objetivos sobre os produtos e processos, para apoiar os objetivos da organização. Tem três camadas: objetivos (o que se quer saber), medidas com definição operacional (o que e como medir) e o repositório consolidado (onde os dados vivem e são analisados).

**O que ele garante.** Que (1) os objetivos de medição derivam dos objetivos de negócio — não se mede por medir; (2) cada medida tem uma definição operacional completa (nome, fórmula, coleta, armazenamento, análise); (3) as medidas são coletadas, verificadas quanto à qualidade e consolidadas num repositório; (4) o desempenho é analisado a partir desse repositório; e (5) resultados geram ações e são comunicados.

**Por que fazer.** Decisão sem dado é palpite. Medição dá base objetiva para estimar (quanto custa o próximo projeto?), para melhorar (onde estamos perdendo tempo?) e para provar (estamos melhorando?). Mas medição mal-feita engana: por isso a definição operacional e a verificação de qualidade são obrigatórias.

**Como usar no dia a dia.**
1. Defina os objetivos de medição a partir dos objetivos de negócio (Plano de Medição, seção 1). Esta é a parte que só a liderança pode fazer.
2. Para cada medida, mantenha a definição operacional completa (Repositório, seção 2).
3. Colete os dados, verifique a qualidade e consolide no Repositório (seção 3).
4. Analise o desempenho a partir do Repositório; gere ações; comunique (Plano, seções 3–5).

**Erros comuns a evitar.**
- **Medir velocidade como se fosse comparável entre times.** Story points são relativos — só comparáveis dentro da própria equipe ao longo do tempo.
- **Tratar benchmark de mercado como meta.** É ponto de partida; a referência real é o histórico próprio.
- **Coletar dado sem definição operacional.** Sem definir como/quando/quem, dois períodos não são comparáveis e o repositório vira lixo consolidado.
