## 1. O fluxo do processo-padrão da Timeware

O ciclo macro de um projeto na Timeware, do fechamento comercial ao encerramento. Use este mapa para se localizar: as seções seguintes detalham o que **você** faz em cada etapa onde seu papel aparece.

```
Faixa 1 — entrada, abertura e requisitos
  1. Originação (comercial) → 2. Escritório de Projetos (passagem de bastão)
  → 3. Abertura (kickoff + TAP) → 4. Discovery + Requisitos (Tech Lead + PO)

Faixa 2 — concepção e aprovação do plano
  5. Concepção (Trilha Projeto + Trilha Produto)
  → 6. Planejamento e aprovação do plano (cliente aprova → baseline)

Faixa 3 — construção e entrega
  7. Grooming → Planning → Dev + code review → QA (V&V) → Homologação
  → Encerramento formal + lições aprendidas
```

---

## 2. Seu papel no geral

Você responde pelo escritório de portfólio e pelo Time de Melhoria Contínua. Garante recursos e treinamento para os processos, conduz a análise crítica trimestral, gerencia o portfólio (capacidade, recursos compartilhados) e usa as medidas para decidir. Você participa de dois processos:

| Processo | Seu envolvimento | Etapa no fluxo |
|---|---|---|
| OSW | Protagonista — governança + portfólio | Organizacional |
| MED | Usuário — decide na análise crítica | Trimestral |

---

## 3. OSW — Gerência Organizacional de Software

Processos de referência: **PRO-OSW-001** (governança), **PRO-OSW-002** (portfólio). Política: **POL-ORG-001**.

### 3.1 Governança de processos

**Input:** Objetivos de negócio. Medidas organizacionais consolidadas (do Responsável de Medição).

**O que você faz:** Garante recursos e treinamento para os processos (PRO-OSW-001 §3). Alinha autoridade, competências e colaboradores aos objetivos (§4). Gere riscos e oportunidades organizacionais (§5). Garante coleta/análise/uso de medidas organizacionais (§6). Conduz a **análise crítica trimestral** que alinha os processos aos objetivos (§7) — você conduz e reporta ao CEO.

**Output:** Diretrizes de processo comunicadas → POL-ORG-001 (Confluence). Decisões e atas da análise crítica trimestral → Confluence. Riscos organizacionais → Jira.

**Onde fica:** Confluence (política, análise crítica); Jira (riscos).

**Atende:** OSW 1 a 7.

### 3.2 Gestão de portfólio

**Input:** Demandas/projetos, quadro de capacidade.

**O que você faz:** Prioriza oportunidades de negócio/investimentos (PRO-OSW-002 §3). Estabelece recursos, orçamento e autoridade do portfólio (§4). Mantém e trata os projetos do portfólio conforme acordos (§5).

**Output:** Decisões de portfólio + quadro de capacidade → Confluence.

**Onde fica:** Confluence.

**Atende:** OSW 8, 9, 10.

---

## 4. MED — Medição (uso)

Plano de referência: **PLA-MED-001** (§5).

**Input:** Indicadores consolidados pelo Responsável de Medição (mensal).

**O que você faz:** Na análise crítica trimestral, usa os indicadores para avaliar o desempenho e decidir ações corretivas (MED 5).

**Output:** Ações corretivas a partir das medidas → registradas na análise crítica / Jira.

**Onde fica:** Confluence (análise crítica) + Jira (ações).

**Atende:** MED 5; OSW 6.

> **Separação de papéis:** você **usa** as medidas para decidir; quem **opera** (coleta, consolida) é o Responsável de Medição. São papéis separados.

---

## 5. O que é medido no seu trabalho

Você não alimenta os 7 indicadores de projeto diretamente. O que você mede é a **efetividade da gestão organizacional** — via análise crítica: as ações tomadas geraram resultado?
